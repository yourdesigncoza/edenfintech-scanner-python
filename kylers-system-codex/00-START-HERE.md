# Start Here: Codex Extrapolation of Kyler's System

This framework is a Codex-built synthesis of the DeepValue wiki (articles + chats + process guides) as of **March 10, 2026**.

It keeps Kyler's core DNA:
- Buy quality businesses in temporary trouble
- Prioritize downside protection over upside fantasy
- Target 30%+ annualized returns for new capital
- Run a concentrated portfolio (about 12 positions max)
- Hold 1-4 years while turnaround catalysts play out

It also adds structure where the wiki is implicit:
- Explicit assumption tracking
- Hard invalidation triggers
- Better separation between `new capital sizing` and `legacy oversized winners`
- A cleaner handoff from valuation -> decision -> portfolio construction

## System Intent

This is a **risk-first deep value turnaround operating system**, not a trading strategy.

Use it when:
- Market narrative is too bearish relative to business reality
- There are identifiable catalysts that can close the perception gap
- You can model a reasonable base case and a survivable downside case

Do not use it when:
- Thesis depends on one binary event
- You cannot articulate why the market is wrong
- There is no catalyst path, only "it's cheap"

## What Changed from Source Material

This Codex version makes 5 practical assumptions:
1. **Score quality beats score precision.** Inputs are subjective, but process consistency is non-negotiable.
2. **Balance sheet survival is the first moat in turnarounds.** Everything else is secondary if runway is short.
3. **Forward returns govern recycling.** Existing winners are candidates for trim/sell when forward CAGR compresses.
4. **Position size is a function of asymmetry, not story quality.** Better story with poor asymmetry gets smaller size.
5. **No catalyst, no position.** Cheapness without a timeline is a trap by default.
