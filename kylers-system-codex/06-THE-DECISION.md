# Step 6: The Decision

## Scoring Inputs

Weighted score components:
- Reasonable worst-case downside (45%)
- Base-case probability (40%)
- Base-case CAGR (15%)

Downside is penalized nonlinearly (curved penalty), consistent with wiki process.

## Three Capital Scenarios

1. Cash available + want theme exposure:
- Compare new idea vs best existing add candidate

2. No cash + want theme exposure:
- Replace weakest holding only if clearly better
- Practical rule from wiki: recycle when existing forward returns fall below ~15% and replacement has superior asymmetry

3. No cash + theme already full:
- Compare only against weakest holding in same theme

## Codex Extrapolation: Replacement Threshold

Use a two-gate replacement test:
- `Gate A`: Forward CAGR delta is meaningful (not marginal)
- `Gate B`: Downside profile is equal or better

If either fails, do not churn.
