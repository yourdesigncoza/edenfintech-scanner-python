    - Test: `hydrate-sector` without sector_name argument prints usage error
    - Test: `sector-status` with no hydrated sectors prints empty table
    - Test: `sector-status` with one hydrated sector prints sector name, hydration date, and FRESH/STALE status
    - Test: `sector-status --sector "Consumer Defensive"` filters to single sector
    - Test: `hydrate-sector` with `--model` flag passes model to GeminiClient
  </behavior>
  <action>
    1. Add two command handler functions to `cli.py`:

    **`_cmd_hydrate_sector(sector_name, sub_sectors, model)`:**
    - Load config via `load_config()`, require `gemini_api_key`
    - Create `GeminiClient(config.gemini_api_key, model=model)` if model provided, else default
    - Call `hydrate_sector(sector_name, sub_sectors=sub_sectors, client=client, config=config)`
    - Print success message with path to knowledge.json
    - Return 0 on success, 1 on error (catch SchemaValidationError, ValueError)

    **`_cmd_sector_status(sector_name_filter)`:**
    - Discover project root via `discover_project_root()`
    - Load registry.json from `data/sectors/registry.json`
    - If sector_name_filter provided, check only that sector via `check_sector_freshness()`
    - Otherwise iterate all sectors in registry, call `check_sector_freshness()` for each
    - Print tabular output: Sector | Hydrated | Age (days) | Status (FRESH/STALE/NOT_HYDRATED)
    - Flag stale sectors (> 180 days)
    - Return 0

    2. Add subparser definitions in `build_parser()` (before `return parser`):

    ```python
    hydrate_sector_p = subparsers.add_parser("hydrate-sector")
    hydrate_sector_p.add_argument("sector_name")
    hydrate_sector_p.add_argument("--sub-sectors", nargs="+", help="Override sub-sector list")
    hydrate_sector_p.add_argument("--model", default=None, help="Gemini model override")

    sector_status_p = subparsers.add_parser("sector-status")
    sector_status_p.add_argument("--sector", default=None, help="Check specific sector")
    ```

    3. Add dispatch in `main()`:
    ```python
    if args.command == "hydrate-sector":
        return _cmd_hydrate_sector(args.sector_name, args.sub_sectors, args.model)
    if args.command == "sector-status":
        return _cmd_sector_status(args.sector)
    ```

    4. Add `TestSectorCli` class to `tests/test_sector.py` with tests for both commands. Mock `GeminiClient` transport to return fixture responses. Use `tempfile.TemporaryDirectory` and patch `discover_project_root` to point to temp dir. Call `main()` directly with argv lists.
  </action>
