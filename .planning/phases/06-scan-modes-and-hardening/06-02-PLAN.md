       _process_single_ticker(ticker, auto_result, *, config, out_dir, judge_transport=None) -> TickerResult:
         - Takes an AutoAnalyzeResult from auto_analyze

       auto_scan(tickers: list[str], *, config: AppConfig, out_dir: Path | None = None, ...) -> ScanResult:
         - Generate scan_id from timestamp
         - Default out_dir to data/scans/<scan_id>/
         - For each ticker: auto_analyze -> _process_single_ticker (sequential, catches errors per ticker)
         - Write manifest, return ScanResult

       sector_scan(sector_name: str, *, config: AppConfig, out_dir: Path | None = None, max_workers: int = 3, excluded_industries: list[str] | None = None, fmp_client: FmpClient | None = None, ...) -> ScanResult:

         - Cluster survivors by "industry" field from screener results
         - Run auto_analyze per ticker using ThreadPoolExecutor(max_workers=max_workers)
         - Process each result through _process_single_ticker
         - Write manifest with scan_type="sector-scan" and sector name
         - Return ScanResult
