  <name>Holding Review Core Functions (TDD)</name>
  <files>src/edenfintech_scanner_bootstrap/holding_review.py, tests/test_holding_review.py</files>
  <behavior>
    HOLD-01 forward_return_refresh:
    - Given base_case {revenue_b=3.0, fcf_margin_pct=10, multiple=24, shares_m=120}, current_price=30.0, years_remaining=2.0
      -> target_price = valuation_target_price(3.0, 10, 24, 120) = 60.0
      -> forward_cagr_pct = cagr_pct(30.0, 60.0, 2.0) = 41.42
    - years_remaining floored at 0.25 to avoid distortion
    - Returns dict with target_price, current_price, forward_cagr_pct, years_remaining

    HOLD-02 thesis_integrity_check:
    - Given triggers=[{trigger: "Margin erosion", evidence: "FCF <5%"}] and current_evidence=[{trigger: "Margin erosion", status: "INVALIDATED", evidence: "Q3 FCF margin 2%"}]
      -> assessment with current_status="INVALIDATED"
    - Given no matching evidence -> status stays "UNCHANGED"
    - Overall status = worst individual status (INVALIDATED > DEGRADED > UNCHANGED > IMPROVED)
    - THESIS_STATUSES = {"IMPROVED", "DEGRADED", "UNCHANGED", "INVALIDATED"}

    HOLD-03 evaluate_sell_triggers:
    - Trigger 1: current_price >= target_price AND forward_cagr < 30 -> TARGET_REACHED_LOW_FORWARD fires
    - Trigger 2: price_gain > 50% from purchase AND forward_cagr < 15 -> RAPID_RERATING_LOW_FORWARD fires
    - Trigger 3: thesis overall_status == "INVALIDATED" -> THESIS_BREAK fires
    - No trigger condition met -> returns empty list
    - Multiple triggers can fire simultaneously

    HOLD-04 replacement_gate:
    - Gate A: replacement_cagr - holding_cagr > 15.0 -> passed
    - Gate B: replacement_downside <= holding_downside -> passed
    - replacement_justified = gate_a AND gate_b
    - Edge: exactly 15pp delta -> gate_a = False (strictly greater than)

    HOLD-05 fresh_capital_weight:
    - Computes floor_price from worst_case, downside from current_price, decision_score, score_to_size_band
    - Returns {fresh_capital_max_weight: band_str, score: float, downside_pct: float}

    review_holding (integration):
    - Calls all 5 functions, returns consolidated dict
    - replacement_candidate is optional (None -> no replacement_gate in result)
    - sell_triggered = len(sell_triggers) > 0
  </behavior>
  <implementation>
    Create holding_review.py with these functions. All financial math delegated to scoring.py imports -- NO reimplementation of valuation formulas.

    Constants:
    - THESIS_STATUSES = {"IMPROVED", "DEGRADED", "UNCHANGED", "INVALIDATED"}
    - THESIS_SEVERITY ordering for "worst wins": INVALIDATED > DEGRADED > UNCHANGED > IMPROVED
    - RAPID_RERATING_THRESHOLD = 50.0 (percent gain from purchase)
    - FORWARD_HURDLE_PCT = 30.0 (sell trigger 1)
    - RAPID_FORWARD_THRESHOLD = 15.0 (sell trigger 2, conservative end of 10-15% range)
    - MIN_YEARS_REMAINING = 0.25 (floor for years_remaining)

    thesis_integrity_check: Match current_evidence entries to invalidation_triggers by trigger text. For unmatched triggers, default to UNCHANGED. Overall = worst status using severity ordering.

    Anti-patterns from research:
    - Do NOT pass price data into thesis_integrity_check (price is NOT a thesis indicator)
    - Do NOT re-fetch raw bundles; use original valuation inputs only
    - years_remaining = max(original_years - elapsed, 0.25) -- caller computes elapsed
  </implementation>
